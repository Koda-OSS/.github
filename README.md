> Open-source libraries, specs, and tooling from [**Koda.sh**](https://koda.sh) — maintained by the [**Koda-sh**](https://github.com/Koda-sh) team and released under open licenses.

**Koda-OSS** is the public home for Koda's open-source work. Proprietary products and infrastructure live under [**Koda-sh**](https://github.com/Koda-sh); libraries, language standards, and documentation are published here for anyone to use, study, and contribute to.

**Perfect for:** Developers building on the Koda ecosystem, contributors looking for specs and reference implementations, and teams evaluating Blit and related open standards.

## Why Koda-OSS Exists

Organization profile READMEs are your front door on GitHub. For **Koda-OSS**, that door leads to:

- **🎯 Open standards** — Publish language specs, parsers, and reference tooling that anyone can adopt without vendor lock-in
- **🚀 Discoverability** — Make it easy for developers and AI tools to find Koda's public libraries and documentation
- **👥 Contributor onboarding** — Point newcomers to docs, Discord, and the right repositories from one place
- **📦 Project showcase** — Highlight active OSS work like [**blit.ts**](https://github.com/Koda-OSS/blit.ts)
- **🤝 Community building** — Connect the GitHub org with [Discord](https://discord.gg/Uc2Dnyb3Ej) and [docs.koda.sh](https://docs.koda.sh)
- **💼 Transparency** — Separate open-source releases ([**Koda-OSS**](https://github.com/Koda-OSS)) from product work ([**Koda-sh**](https://github.com/Koda-sh))

## How to Get Started

**To use our libraries:**

1. Browse [Projects](#projects) and open the repository you need
2. Read the package README and follow its install instructions
3. Open the [documentation hub](https://docs.koda.sh) for guides and API references
4. For Blit specifically, start at [Blit docs](https://docs.koda.sh/?alias=blit)

**To contribute:**

1. Pick a repository and read its contribution guidelines
2. Open an issue or discussion before large changes when possible
3. Join the [Discord](https://discord.gg/Uc2Dnyb3Ej) for questions and community support
4. Follow documented behavior in `koda/docs/` when contributing to specs or parsers

## Contributing

We welcome issues, documentation improvements, and pull requests across our public repositories.

**Before you open a PR:**

- [ ] Read the target repository's README and contribution notes
- [ ] For spec or parser changes, check [docs.koda.sh](https://docs.koda.sh) first — docs define expected behavior
- [ ] Keep changes focused and well-described in the PR summary
- [ ] Ask questions in [Discord](https://discord.gg/Uc2Dnyb3Ej) if you're unsure where a change belongs

**Other ways to help:**

- Report bugs with clear reproduction steps
- Improve examples and documentation
- Share feedback on APIs and language design

## FAQ

### What is Koda-OSS?

**Koda-OSS** is Koda's open-source GitHub organization. It hosts public libraries, specs, and tooling such as [**blit.ts**](https://github.com/Koda-OSS/blit.ts).

### How is Koda-OSS different from Koda-sh?

[**Koda-sh**](https://github.com/Koda-sh) is the non-OSS organization for products and infrastructure. [**Koda-OSS**](https://github.com/Koda-OSS) is where open-source releases live.

### Where is the documentation?

The documentation hub is at [**docs.koda.sh**](https://docs.koda.sh). Blit-specific docs are at [**docs.koda.sh/?alias=blit**](https://docs.koda.sh/?alias=blit).

### Where can I get help?

Join the official [**Discord**](https://discord.gg/Uc2Dnyb3Ej) or open an issue in the relevant repository.

### Where is this README defined?

This organization profile README lives in [`.github/profile/README.md`](https://github.com/Koda-OSS/.github/blob/main/profile/README.md) and is rendered on the [Koda-OSS overview page](https://github.com/Koda-OSS).
